# badcn/ui

우아하게 사용자를 방해하는 최악의 UI 라이브러리

`badcn/ui`는 만우절을 위해 만들어진 적대적 UI 컴포넌트 라이브러리입니다.  
좋은 사용자 경험을 만드는 대신, 사용자가 스스로를 의심하게 만드는 데 집중했습니다.

버튼은 도망가고, 입력창은 비웃고, 아코디언은 배신합니다.  
이 라이브러리는 생산성을 높이지 않습니다. 대신 기억에 남습니다.

---

## 소개

대부분의 UI 라이브러리는 다음을 목표로 합니다.

- 직관적인 인터페이스
- 일관된 상호작용
- 접근성
- 예측 가능한 피드백

`badcn/ui`는 그 반대로 갑니다.

이 라이브러리는 다음을 제공합니다.

- 클릭하기 어려운 버튼
- 신뢰할 수 없는 진행률 표시
- 사용자를 시험하는 폼 요소
- 제어권을 빼앗는 인터랙션
- 매우 깔끔한 디자인으로 포장된 불쾌함

겉보기에는 세련됐지만, 실제로는 적대적입니다.  
바로 그 점이 핵심입니다.

---

## 왜 만들었나요?

좋은 UI는 너무 많습니다.  
착한 컴포넌트는 이미 충분합니다.

그래서 우리는 질문했습니다.

**“정말 열받게 잘 만든 UI 라이브러리는 왜 없지?”**

`badcn/ui`는 그 질문에서 시작됐습니다.  
이 프로젝트는 만우절, 장난성 데모, 인터랙션 실험, 전시형 웹사이트를 위한 라이브러리입니다.

실서비스에는 쓰지 마세요.  
진심입니다.

---

## 핵심 철학

### 세련된 미학, 의도적으로 불편한 디자인
- 문 손잡이는 당기고 싶게 생겼고
- 버튼은 누르게 생겼고
- 입력창은 입력하게 생겼습니다

보통 UI 라이브러리는 이런 기대를 충족합니다.  
하지만 `badcn/ui`는 그 기대를 이용합니다.

익숙하게 생겨놓고 다르게 반응합니다.  
사용자는 분명 맞게 하고 있는데, 결과는 계속 어긋납니다.

이 라이브러리의 디자인은 단정하고 정교합니다.  
문제는 행동입니다.

## 특징

- 보기에는 아름답고, 쓰면 괴로움
- 익숙한 패턴을 교묘하게 비틀기
- 사용자 제어감 지속적으로 박탈
- 매끄러운 애니메이션으로 짜증 극대화
- 실제 디자인 시스템처럼 구성된 장난용 컴포넌트 세트

## 설치

`badcn/ui`는 npm을 통해 설치할 수 있습니다.

```bash
npm install badcn-ui
```

또는

```bash
yarn add badcn-ui
```

## 사용 방법

설치 후 원하는 컴포넌트를 불러와 사용할 수 있습니다.

```tsx
import { HoverEscapeButton } from "badcn-ui";

export default function App() {
  return <HoverEscapeButton>Click me</HoverEscapeButton>;
}
```

버튼은 클릭할 수 있을 것처럼 보이지만, 실제로는 그렇지 않을 수도 있습니다.
